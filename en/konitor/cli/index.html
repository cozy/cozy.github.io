Test¶

You can test a konnector with slug or repository name:

  • konitor test trainline
  • konitor test cozy-konnector-trainline

Help¶

$ konitor test --help

_                      _   _
| | __   ___    _ __   (_) | |_    ___    _ __
| |/ /  / _ \  | '_ \  | | | __|  / _ \  | '__|
|   <  | (_) | | | | | | | | |_  | (_) | | |
|_|\_\  \___/  |_| |_| |_|  \__|  \___/  |_|

vX.X.X

konitor test <name> [options]

Test a konnector

Options:
--help             Show help                                         [boolean]
--version          Show version number                               [boolean]
--config, -c       Path to a config file                      [default: false]
--interactive, -i  Launch interactive mode                     [default: true]

check ü§ì¶

Check is used to programmatically check our guidelines against a repository. Use it before publishing your konnector for it to be production ready üöÄ

Here is a sample output :

$ konitor check ./cozy-konnector-ameli/

## Checking ../cozy-konnector-ameli/

* Eslint with prettier is used to lint the code (check for eslintConfig in package.json) ‚ùå
  - eslintConfig should extend from prettier ‚ùå
  Check the documentation:  https://github.com/konnectors/docs/blob/master/status.md#linting

* Fields (necessary for login) are defined in manifest.konnector ‚ùå
  - The fields should not be in old format ‚ùå

* Travis is correctly configured to deploy master/prod. ‚úÖ

* Renovate should be correctly configured. ‚ùå
  - Renovate config should extend from the cozy-konnector config ‚ùå
  Check the documentation:  https://github.com/konnectors/docs/blob/master/status.md#renovate

* Repository should have 4 branches. ‚úÖ

You can check multiple repositories :

$ konitor check ./cozy-konnector-ameli/ ./cozy-konnector-orange/
...
